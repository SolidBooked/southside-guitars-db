## 2. Confirmation: Pre-2020 StockID=NULL Rows in tblSaleItem

**Question from Session 3:** Why do 8,869 rows of tblSaleItem have StockID=NULL or orphaned values pre-August 2020?

**Resolution:** [V]

**PawnIt → CWServer Migration (August 2020) duplicated second-hand stock:**

The migration from the legacy PawnIt system to ProCreate's CWServer duplicated roughly 11,000 second-hand items into approximately 22,000 records. Each original item produced two rows in the target system:

1. **Numerical StockID** — the correct CWServer second-hand record, linked to tblTranItems
2. **INV-prefix StockID** — incorrectly typed as new supplier stock (e.g., INV-001, INV-002)

**Why this matters:**
- The INV-prefix duplication accounts for the StockID=NULL and orphaned rows seen in pre-Aug 2020 tblSaleItem data
- INV-prefix items carry no QLD Second Hand Dealers Act tracking obligation (they are nominally supplier stock, not second-hand goods)
- **Critical distinction:** Post-August 2020, INV prefix in tblSaleItem.StockID is **legitimate and current** — it denotes new stock received from supplier invoices, not a migration artefact
- The duplication does not affect reconciliation — both rows reflect real sales; the INV-prefix copy just lacks the second-hand tracking metadata

**Historical context:** PawnIt was the legacy pawnbroker management system used 2013–July/August 2020. CWServer (ProCreate) went live in August 2020 with a one-time data migration that preserved all transaction dates.

## 3. PayType Code Map — Fully Verified (All [V])

**Method:** User shared a screenshot of the PosWiz Point of Sale payment screen showing all payment method buttons in their display order.

**Key insight:** PosWiz payment buttons are **0-indexed sequentially from the top** — position 1 (first button visible) = Cash = PayType 0. This order directly maps to the PayType integer codes stored in tblPayments.

**Fully resolved PayType map — all codes verified [V]:**

| PayType | Count | Method | Notes |
|---------|-------|--------|-------|
| 0 | 7,464 | Cash | Tendered column always populated; distinguishes from all other types |
| 1 | 2 | Cheque | Legacy button; 2 rows total; almost never used |
| 2 | 22,990 | EFTPOS / Card (all brands) | Tendered=NULL; single integrated disbursement via First Data (FDSMA) provider |
| 3 | 142 | AMEX button (PosWiz UI artefact) | Separate button in PosWiz; used by untrained staff mid-2021–Aug 2025; settles in same EFTPOS batch as PayType 2; treat as PayType 2 for reconciliation |
| 4 | 3 | VISA (pre-integration only) | Dec 2020 only; pre-EFTPOS-integration; consolidated under PayType 2 from mid-2021 |
| 5 | 99 | MasterCard (pre-integration only) | Pre-Jul 2021; pre-EFTPOS-integration; consolidated under PayType 2 from mid-2021 |
| 6 | 463 | Direct bank transfer / EFT | Max $11,000; appears on loans (3 rows) and refunds (49 rows) |
| 7 | 102 | Credit note / gift voucher | Mar 2022–present; no bank movement; Xero account 808 (CURRLIAB); manual journal entry required |
| 8 | 1,710 | ALL online orders | PayPal + Shopify Payments + Afterpay + ZipPay via single "PayPal" button in PosWiz; no gateway granularity in DB; disambiguation requires paypal_sales_extract.py |
| 9 | 3 | Other credit card | Rare; 3 rows total |

**EFTPOS integration timeline (historical context):**
- **Pre-mid-2021:** VISA (PayType 4), MasterCard (PayType 5), and AMEX (PayType 3) each had their own dedicated PosWiz button
- **Mid-2021 onwards:** First Data (FDSMA) integrated provider consolidated all card brands under PayType 2. PayType 4 and 5 stopped being used. PayType 3 (AMEX) persisted due to staff habit but routes through the same settlement batch.

**ANZ bank settlement by PayType (corrected):**
- PayType 0 (Cash) → Cash on Hand account (no ANZ line per transaction)
- PayType 2+3 (EFTPOS/AMEX) → First Data / FDSMA single daily settlement batch
- PayType 6 (Bank Transfer) → Direct ANZ transfer (appears as "Other" in banking summary)
- PayType 7 (Credit note/Voucher) → NO bank movement; manual journal only (Xero account 808)
- PayType 8 (Online/"PayPal") → Four separate ANZ streams: TRANSFER FROM PAYPAL, TRANSFER FROM SHOPIFY, TRANSFER FROM AFTERPAY, TRANSFER FROM ZIPMONEY
